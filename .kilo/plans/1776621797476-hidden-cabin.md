# Plan: Autenticación y Estructura de Login

## Objetivo
Implementar el sistema de autenticación del usuario guardando datos en secure-storage y crear la estructura de acciones para los endpoints.

---

## 1. Modificar `useAuthStore` (zustand)

**Archivo:** `src/store/useAuthStore.ts`

### Cambios:
- Expandir la interfaz `AuthState` para incluir:
  ```typescript
  interface AuthState {
    isLoggedIn: boolean;
    isHydrated: boolean;
    firstName: string | null;
    lastName: string | null;
    userType: string | null;
    token: string | null;
    email: string | null;
    login: (email: string, password: string) => Promise<boolean>;
    logout: () => Promise<void>;
    checkAuthStatus: () => Promise<void>;
  }
  ```

- Modificar la función `login` para:
  1. Llamar al action `loginAction` con email y password
  2. Si el login es exitoso, guardar todos los datos en secure-storage:
     - `firstName`, `lastName`, `userType`, `token`, `email`
  3. Actualizar el estado con los datos del usuario

- Modificar `logout` para limpiar todos los datos de secure-storage

- Modificar `checkAuthStatus` para cargar todos los datos al iniciar la app

---

## 2. Crear Carpeta `core/actions`

**Estructura:**
```
core/
├── actions/
│   ├── login.action.ts
│   └── index.ts
```

### `login.action.ts`:
- Función que hace fetch al endpoint `/auth/login`
- Recibe `email: string` y `password: string`
- Retorna el objeto con los datos del usuario o lanza error

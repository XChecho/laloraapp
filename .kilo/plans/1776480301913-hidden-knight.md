# Plan: Crear Archivo .md para Backend NestJS

## Objetivo
Crear un nuevo archivo de especificaciones `backend_init_nestjs.md` para construir el backend de Laloraapp con **NestJS** (Node.js/TypeScript) en lugar de .NET.

## Proyecto Identificado
**Laloraapp** - Aplicación móvil React Native/Expo para restaurante/bar con canchas deportivas.

**Roles del sistema:** Admin, Cashier, Kitchen, Waitress, CanchaManager

**Dominios:**
1. Identity & Access (usuarios, login, roles)
2. Catalog/Menú (categorías, productos, precios, modificadores)
3. Orders (creación, estados de pedido, cocina)
4. Tables & Canchas (gestión de mesas y canchas deportivas)
5. Reservations (reservas de canchas)
6. Billing/Caja (cierres, pagos, reportes)

## Adaptaciones para NestJS

| Aspecto | .NET (actual) | NestJS (nuevo) |
|---------|---------------|----------------|
| Lenguaje | C# 12+ | TypeScript |
| Framework | ASP.NET Core | NestJS |
| DB | PostgreSQL + EF Core | PostgreSQL + Prisma/TypeORM |
| Tiempo Real | SignalR | Socket.io |
| Validación | FluentValidation | class-validator |
| CQRS | MediatR | Servicios + DTOs NestJS |
| Docs | Swagger/Scalar | Swagger |
